Publicación en Servidores

Guía técnica completa para la migración de Apache a Flask utilizando Proxy Inverso, Gunicorn y Systemd.

1. Tipos de Servidor

Apache (Puerto 80)

Servidor multipropósito. Su función principal es servir archivos estáticos y aplicaciones bajo el Stack LAMP (PHP). Su punto de montaje estándar es /var/www/html.

Uso: Contenido General

Flask (Puertos Altos)

Micro-framework de Python. Opera en puertos específicos (5000, 5001...). Dado que el usuario final no suele acceder por estos puertos, es obligatorio el uso de un Proxy Inverso.

127.0.0.1:5000

2. Virtualhost de jocarsa.com

Configuración

Permite que el servidor identifique qué carpeta servir según el dominio solicitado:

/etc/apache2/sites-available/jocarsa.com.conf 
<VirtualHost *:80>
    ServerName jocarsa.com
    ServerAlias www.jocarsa.com
    DocumentRoot /var/www/html/jocarsa
    <Directory /var/www/html/jocarsa>
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>
sudo a2ensite jocarsa.com.conf
sudo apachectl configtest
sudo systemctl reload apache2

3. Subdominio colores.jocarsa.com

Mismo patrón para segmentar servicios adicionales en subdirectorios independientes.

<VirtualHost *:80>
    ServerName colores.jocarsa.com
    DocumentRoot /var/www/html/colores
</VirtualHost>

Recordatorio DNS: Es imperativo crear un registro A o CNAME en el panel de control del dominio que apunte "colores" a la IP del VPS.

4. Apache como Proxy Inverso

Puente lógico entre el puerto 443 (HTTPS seguro) y la ejecución interna de Flask en el puerto 5000.

Input

Usuario :443

Shield

Apache VHost

Engine

gunicorn (5000)

App

miniservidor.py

Habilitación de módulos:

sudo a2enmod proxy proxy_http headers && sudo systemctl restart apache2
/etc/apache2/sites-available/flask.jocarsa.com.conf
<VirtualHost *:443>
    ServerName flask.jocarsa.com
    SSLEngine on
    SSLCertificateFile /etc/apache2/ssl/jocarsa_combined.cer
    SSLCertificateKeyFile /etc/apache2/ssl/jocarsa.key

    ProxyPreserveHost On
    RequestHeader set X-Forwarded-Proto "https"
    ProxyPass / http://127.0.0.1:5000/
    ProxyPassReverse / http://127.0.0.1:5000/
</VirtualHost>

5. Flask como servicio (Systemd + Gunicorn)

Archivo: miniservidor.py

from flask import Flask
from werkzeug.middleware.proxy_fix import ProxyFix

app = Flask(__name__)
# Crucial para detectar HTTPS detrás del proxy
app.wsgi_app = ProxyFix(app.wsgi_app, x_for=1, x_proto=1, x_host=1)

@app.route("/")
def home():
    return "Servidor Operativo con SSL"

Archivo: /etc/systemd/system/flask_app.service

[Unit]
Description=Servicio Flask para flask.jocarsa.com
After=network.target

[Service]
User=www-data
Group=www-data
WorkingDirectory=/home/miniservidor/
Environment="PATH=/home/miniservidor/venv/bin"
ExecStart=/home/miniservidor/venv/bin/gunicorn --workers 2 --bind 127.0.0.1:5000 miniservidor:app
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload sudo systemctl enable flask_app sudo systemctl start flask_app
