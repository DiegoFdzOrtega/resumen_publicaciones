🚀 Infraestructura de Red Jocarsa (Nodo 026) Este proyecto consiste en el despliegue de una arquitectura de servidor profesional que combina la robustez de Apache con la flexibilidad de Flask (Python). 🤔 ¿Qué es esto? Es un sistema de Proxy Inverso. En lugar de exponer nuestra aplicación Python directamente a Internet (lo cual es peligroso e ineficiente), colocamos a Apache como un "escudo" o intermediario que gestiona todo el tráfico entrante. ⚙️ ¿Qué hace el sistema? El flujo de una petición funciona de la siguiente manera: Recepción: El usuario hace una petición a flask.jocarsa.com. Filtrado (Apache): Apache recibe la conexión, gestiona el cifrado SSL/HTTPS y comprueba que la petición es legítima. Puente (Proxy): Apache le pasa la petición "limpia" a nuestra app de Flask que corre internamente en el puerto 5000. Procesamiento (Flask): Python ejecuta la lógica y le devuelve el resultado a Apache. Respuesta: Apache entrega el contenido final al usuario. 🎯 ¿Para qué funciona? (Ventajas) Seguridad: Flask nunca está en contacto directo con los ataques de Internet. Rendimiento: Apache gestiona los archivos estáticos (imágenes, CSS) de forma mucho más rápida que Python. Estabilidad: Gracias a Systemd, si la aplicación de Python falla, el sistema la reinicia automáticamente sin intervención humana. Centralización: Permite tener varios subdominios (colores.jocarsa.com, jocarsa.com, etc.) conviviendo en la misma máquina de forma organizada. 🛠️ Especificaciones Técnicas 1. Servidor Perimetral (Apache) Ubicado en /etc/apache2/sites-available/. Modo: Proxy Inverso (mod_proxy). SSL: Configurado para encriptación de datos en tránsito. 2. Motor de Aplicación (Flask + Gunicorn) Ubicado en /home/miniservidor/. WSGI: Usamos Gunicorn para permitir que Flask maneje múltiples peticiones a la vez. Middleware: Implementación de ProxyFix para que Flask sepa la IP real del usuario y no la del proxy. 3. Gestión de Procesos (Systemd) Servicio: flask_app.service Función: Garantiza que el proceso miniservidor:app esté siempre en ejecución. 🚀 Comandos Rápidos Bash # Ver si todo está vivo systemctl status apache2 systemctl status flask_app # Ver los logs en tiempo real (vital para depurar) journalctl -u flask_app -f Proyecto Intermodular DAM | Ingeniería de Sistemas Jocarsa 2026
